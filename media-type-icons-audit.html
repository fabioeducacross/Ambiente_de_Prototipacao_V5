Conclusões da auditoria em produção

  • check_circle
    3 de 5 tipos têm ícone confirmado no código de produção — sports_esports (Jogos), quiz (Questões) e book_5 (Livros) foram encontrados em componentes reais (MissionReportGames.vue, teacher.js e GeneralDashboard.vue).
  • error
    Vídeos e Músicas não possuem ícone mapeado em produção — as abas na Educateca (LibraryRoot.vue) são definidas apenas com { title: 'Vídeos' } e { title: 'Músicas' }, sem propriedade icon. Os componentes MeetVideos.vue e MeetMusics.vue não usam nenhum Material Symbol para representar o tipo de mídia.
  • warning
    Não existe um mediaTypeIconMap centralizado em produção — cada contexto define seu próprio ícone de forma ad hoc. Recomendado criar o mapa centralizado abaixo e adotá-lo tanto no protótipo quanto ao implementar em produção.
  • info
    Sugestão para Vídeos: smart_display e para Músicas: music_note — semanticamente adequados e disponíveis no Material Symbols Outlined. Adotar no protótipo até que produção defina oficialmente.

Código sugerido — mediaTypeIconMap centralizado

// FrontOffice/src/shared/mediaTypeIconMap.js
// Mapa canônico de ícones por tipo de mídia — Material Symbols Outlined
// Fontes: ✓ confirmado em produção (migration-phase2) | ⚠ sugestão para protótipo

export const mediaTypeIconMap = {
  // ✓ MissionReportGames.vue · MediaCard icon="sports_esports"
  jogos:    { icon: 'sports_esports',  color: '#FF9F43', tone: 'warning' },
  // ✓ navigation/vertical/teacher.js · { title: 'Avaliações', icon: 'quiz' }
  questoes: { icon: 'quiz',             color: '#7367F0', tone: 'primary' },
  // ✓ GeneralDashboard.vue + teacher.js · icon: 'book_5'
  livros:   { icon: 'book_5',           color: '#7367F0', tone: 'primary' },
  // ⚠ sem ícone em produção — sugestão para o protótipo
  videos:   { icon: 'smart_display',   color: '#00CFE8', tone: 'info'    },
  // ⚠ sem ícone em produção — sugestão para o protótipo
  musica:   { icon: 'music_note',      color: '#28C76F', tone: 'success' },
}

// Helper: retorna o ícone para um mediaType, com fallback
export const getMediaTypeIcon = (mediaType) =>
  mediaTypeIconMap[mediaType] ?? { icon: 'category', color: '#B9B9C3', tone: 'muted' }
